# Development Playbook Package

Use this structure when turning source materials into a practical development handbook.

## Package Outline

1. Purpose and audience
2. Source materials used
3. Scope and expected outcome
4. Global prerequisites and environment
5. Phase map or task sequence
6. Task catalog
7. Acceptance and evidence expectations
8. Optional extension or production path
9. Risks and open questions

## Task Card Template

Each task should include:

- Task title
- Goal
- Suggested difficulty or effort
- Inputs and prerequisites
- Concrete actions
- Acceptance checks
- Evidence to submit or keep
- Required references
- Optional references
- Pitfalls or notes

## Quality Rules

- Make acceptance observable: command output, page access, response shape, screenshot, exported file, or other concrete proof.
- Keep dependency order explicit whenever one task relies on earlier setup.
- Preserve sample commands, URLs, paths, and artifact names from the source materials when they are trustworthy.
- Separate required tasks from optional, stretch, or production-only tasks.
- Keep the handbook usable by someone who did not participate in the original design discussion.
