# Source Material Patterns

## PRD or Feature Brief

Extract:

- business goal
- actors and permissions
- success criteria
- scope and exclusions
- explicit rules and edge cases

## PDF Design or Development Docs

Extract:

- module boundaries
- workflows
- state transitions
- non-functional constraints
- technical assumptions

## Spreadsheet Materials

Look for:

- endpoint inventories
- status code tables
- sample payloads
- field naming conventions
- data dictionary patterns

## API Examples

Extract:

- request and response envelope
- auth model
- pagination shape
- field naming conventions
- error semantics

## Cross-Source Reconciliation

- Prefer explicit written rules over isolated examples.
- Prefer repeated patterns over one-off samples.
- Preserve conflicts as conflicts when sources disagree.
